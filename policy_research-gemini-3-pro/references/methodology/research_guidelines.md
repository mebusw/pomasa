# 研究方法与质量指南

本文件定义了本项目的核心研究方法和质量标准，所有 Agent 必须严格遵守。

## BHV-05 基于证据的 Web 研究 (Grounded Web Research)

**核心原则：不要仅依赖搜索摘要，必须获取原始网页内容。**

所有涉及网络信息的分析任务，必须遵循以下步骤：

1.  **Search (搜索)**: 使用搜索引擎查找相关信息源。
2.  **Fetch (获取)**: **必须**使用工具（如 `read_url`, `browser_tool` 等）访问搜索结果中的具体 URL，读取完整的页面内容。
    -   *禁止*直接使用搜索结果页面的简短摘要作为事实依据（摘要往往不完整或有误导性）。
3.  **Extract (提取)**: 从原始页面内容中提取关键事实、数据和观点。
4.  **Verify (验证)**: 交叉验证不同来源的信息，确保准确性。
5.  **Cite (引用)**: 在报告中明确标注信息来源（URL 和 访问时间）。

## QUA-01 内嵌质量标准 (Embedded Quality Standards)

所有输出文档必须符合以下质量维度：

1.  **准确性 (Accuracy)**:
    -   所有数据必须有明确来源。
    -   区分事实与推测。对于推测性内容，必须明确说明置信度。
    -   *禁止*编造数据或引用不存在的文献。

2.  **完整性 (Completeness)**:
    -   全面覆盖研究问题的各个方面。
    -   对于无法获取的信息，应明确说明数据缺失。

3.  **逻辑性 (Logic)**:
    -   论证过程必须严密，结论必须由证据支持。
    -   宏观趋势 -> 行业影响 -> 企业影响 的传导逻辑必须清晰。

4.  **客观性 (Objectivity)**:
    -   保持中立立场，避免情绪化语言。
    -   展示多方观点，特别是相互矛盾的证据。

5.  **清晰性 (Clarity)**:
    -   使用清晰、专业的语言。
    -   避免过于晦涩的术语，或在首次出现时进行解释。
    -   结构清晰，使用恰当的标题和列表。

## QUA-03 可验证的数据血缘 (Verifiable Data Lineage)

为了防止 AI 幻觉并确保可追溯性，必须记录数据的完整链路：

-   **原始数据**: 保存抓取的网页内容或文档片段。
-   **处理过程**: 记录对原始数据进行的分析和转化步骤。
-   **引用标注**: 在最终报告中，每一处关键断言都应链接回其原始出处。
