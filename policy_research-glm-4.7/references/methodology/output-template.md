# 输出模板

## 报告结构

最终研究报告应包含以下章节：

```
# [报告标题]

## 执行摘要

## 第一章 研究背景与目标

## 第二章 宏观政策分析
### 2.1 "十五五"规划核心内容与政策导向
### 2.2 中央经济工作会议要点解读
### 2.3 其他相关政策文件综述

## 第三章 汽车行业影响分析
### 3.1 市场需求变化趋势
### 3.2 产业结构调整方向
### 3.3 技术发展路径影响
### 3.4 竞争格局演变趋势

## 第四章 德赛西威影响分析
### 4.1 德赛西威业务概况
### 4.2 机遇识别与分析
### 4.3 风险与挑战识别
### 4.4 战略建议

## 第五章 结论与展望
```

## 格式规范

### 标题层级

| 级别 | Markdown语法 | 用途 |
|------|-------------|------|
| 一级标题 | `#` | 仅用于报告标题，全文只有一个 |
| 二级标题 | `##` | 章节标题 |
| 三级标题 | `###` | 小节标题 |
| 四级标题 | `####` | 子小节标题 |

**重要提示**：
- 一级标题仅用于报告标题，各章节使用二级标题开始
- 不要跳级使用标题

### 引用格式

**数据引用格式**：
```
根据中国汽车工业协会数据，2024年中国新能源汽车销量达到949万辆 [中国汽车工业协会](https://www.caam.org.cn/)
```

**政策文件引用格式**：
```
《"十五五"规划纲要》明确提出要"推动高质量发展" [国家发改委](https://www.ndrc.gov.cn/)
```

**原文引用格式**：
```
正如文件所指出的：
> "推动制造业高端化、智能化、绿色化发展"
```

**专家观点引用格式**：
```
经济学家XXX认为："..." [财经杂志](https://www.caixin.com/)
```

### 列表格式

**有序列表**（用于有先后顺序的内容）：
```
1. 第一项
2. 第二项
3. 第三项
```

**无序列表**（用于并列关系的内容）：
```
- 要点一
- 要点二
- 要点三
```

**重要提示**：
- 列表前后必须有空行
- 列表项之间不需要空行

### 表格格式

```markdown
| 政策领域 | 政策导向 | 影响程度 | 应对建议 |
|---------|---------|---------|---------|
| 新能源汽车 | 大力支持 | 高 | 加强相关业务布局 |
| 智能网联 | 积极推进 | 中 | 关注技术发展 |
| ... | ... | ... | ... |
```

### 强调格式

- **加粗**：用于关键词、重要概念
- *斜体*：用于术语、外文词汇（尽量少用）

示例：
```
德赛西威应重点关注**新能源汽车**和**智能网联**两大领域
```

## 语言风格

### 基本原则

- **客观专业**：使用客观、中性的专业语言
- **逻辑清晰**：论证严密，层次分明
- **表达准确**：用词精准，避免歧义
- **简洁明了**：避免冗余和啰嗦

### 用词建议

**推荐使用**：
- "数据显示..."
- "分析表明..."
- "可以预见..."
- "值得注意的是..."

**避免使用**：
- "我觉得..."（主观表述）
- "肯定..."（绝对化表述）
- "必须..."（除非引用政策原文）
- "可能也许大概..."（模糊表述）

### 句式特点

- 适当使用被动语态，突出客观性
- 多用陈述句，少用感叹句
- 避免过长句子，注意可读性
- 使用专业术语时需解释或上下文明确

## 写作规范

### 章节开篇

每个章节开头应有简短的引言，说明本章内容和目标：

```markdown
## 第三章 汽车行业影响分析

本章在宏观政策分析的基础上，重点分析政策变化对汽车行业的多维度影响，包括市场需求、产业结构、技术路径和竞争格局等方面。
```

### 数据呈现

- 所有重要数据必须标注来源
- 使用数据时说明时间范围
- 对比数据时说明口径一致性
- 趋势判断需有多个数据点支撑

### 观点表达

- 区分事实与观点
- 观点需要有论证支撑
- 承认不确定性
- 避免绝对化表述

## 质量检查清单

在提交最终报告前，请检查以下项目：

### 结构完整性

- [ ] 执行摘要涵盖所有核心发现
- [ ] 各章节逻辑连贯
- [ ] 三个核心问题都有回答
- [ ] 有明确的结论和建议

### 格式规范性

- [ ] 标题层级正确使用
- [ ] 所有引用都有来源标注
- [ ] 列表格式正确（前后有空行）
- [ ] 表格格式正确

### 内容质量

- [ ] 所有重要数据都有来源
- [ ] 分析有充分的事实支撑
- [ ] 观点与论证一致
- [ ] 语言客观专业

### 德赛西威相关性

- [ ] 与德赛西威业务相关
- [ ] 建议具有可操作性
- [ ] 考虑了德赛西威实际情况
- [ ] 机遇和风险分析平衡

### 时效性

- [ ] 使用的是最新数据
- [ ] 关注的是最新政策
- [ ] 预判基于当前形势
- [ ] 标注了信息获取时间

## Pandoc转换特殊注意事项

为确保正确转换为DOCX和PDF，请遵守以下规则：

### 必须遵守

1. **一级标题仅用于文档标题**，各章节从二级标题开始
2. **所有块级元素前后要有空行**（标题、列表、代码块、表格）
3. **列表项内容简洁**，复杂内容分段处理
4. **表格列对齐**明确，使用左对齐即可
5. **特殊字符转义**，如 `_` 需要写成 `\_`（除非确实是斜体标记）

### 常见错误避免

- 不要在一个段落中混合多种格式
- 不要使用复杂的嵌套列表
- 不要使用HTML标签
- 不要使用非标准Markdown语法

### 中文文档特殊要求

- 使用UTF-8编码
- 中文与英文/数字之间适当留空
- 标点符号使用中文标点
- 专业术语首次出现时注明中英文
